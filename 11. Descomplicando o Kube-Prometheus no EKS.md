<h2 align="center"> Descomplicando o Kube-Prometheus no EKS. </h2>

<h3 align="center"> 11.1.	O que é o Prometheus Operator e o Kube-Prometheus. </h3>

**Prometheus Operator.**  
O Prometheus Operator é uma extensão do Kubernetes que facilita a implantação, o gerenciamento e a configuração do Prometheus e de componentes relacionados (como Alertmanager, Grafana, etc.) em clusters Kubernetes, de forma declarativa.

**Kube-Prometheus.**  
O kube-prometheus é um conjunto de manifestos do Kubernetes que nos permite ter o Prometheus Operator, Grafana, AlertManager, Node Exporter, Kube-State-Metrics, Prometheus-Adapter instalados e configurados de forma tranquila e com alta disponibilidade. Além disso, ele nos permite ter uma visão completa do nosso cluster de Kubernetes. Ele nos permite monitorar todos os componentes do nosso cluster de Kubernetes, como por exemplo: kube-scheduler, kube-controller-manager, kubelet, kube-proxy, etc.

<h3 align="center"> 11.2. Instalando o EKSCTL e o AWS CLI. </h3>

O **EKS (Elastic Kubernetes Service)** é o serviço gerenciado de Kubernetes da AWS. Ele permite que você crie, configure e escale clusters Kubernetes sem precisar instalar e manter manualmente o plano de controle (control plane) ou componentes de gerenciamento.

**O que é eksctl?**  
O eksctl é uma ferramenta CLI simples para criar e gerenciar clusters no EKS, o serviço Kubernetes gerenciado da Amazon para EC2. Para ver mais detalhes, acesse a página do projeto: **[https://eksctl.io](https://eksctl.io/)**

**Download e instalação.**  
Acesse a página **[Installation](https://docs.aws.amazon.com/eks/latest/eksctl/installation.html)** e siga os passos abaixo:  
* No Linux, crie as variáveis:  
  * ARCH=amd64  
  * PLATFORM=$(uname -s)_$ARCH  

* **Baixe o pacote**: curl -sLO "https://github.com/eksctl-io/eksctl/releases/latest/download/eksctl_$PLATFORM.tar.gz"  
* **Extrair os arquivos**: tar -xzf eksctl_$PLATFORM.tar.gz -C /tmp && rm eksctl_$PLATFORM.tar.gz  
* **Mova os arquivos**: sudo mv /tmp/eksctl /usr/local/bin  

Após estes passos, a ferramenta EKSCTL está instalada no sistema:

**O que é AWS CLI?**  
O AWS Command Line Interface (CLI) é uma ferramenta unificada para gerenciar seus serviços da AWS. Com apenas uma ferramenta para baixar e configurar, você pode controlar vários serviços da AWS na linha de comando e automatizá-los por meio de scripts.

A AWS CLI é uma ferramenta unificada que fornece interface consistente para interagir com todas as partes da AWS. **[Acesse Referência de comandos AWS CLI](https://docs.aws.amazon.com/cli/latest/)**.

Acesse a página **[Instruções de instalação e atualização do AWS CLI](https://docs.aws.amazon.com/cli/latest/userguide/getting-started-install.html)**.  

**Download e instalação.**  
Para instalação no sistema operacional Linux, seguir os passos abaixo.  

* **Baixe o pacote**: curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"  
* **Extrair os arquivos**: unzip awscliv2.zip  
* **Instalar o pacote**: sudo ./aws/install  

Para ver se o pacote foi instalado corretamente, execute: **aws --version**

<h3 align="center"> 11.5.  Acessando o nosso Grafana e os Dashboards. </h3>

Para tornar o Grafana acessível, devemos utilizar um Port-Forward no nosso cluster Kubernetes.  

* **Para visualizar os serviços criados, execute**: kubectl get services -A  
* **Para criar o Port-Forward para o Kubernetes**: kubectl port-forward -n monitoring svc/grafana 33300:3000  

**Obs**: como o meu servidor do laboratório não tem interface gráfica, foi necessário criar um túnel para acessar o Grafana na porta 33300.  
* **No host local, execute**: ssh -L 33300:localhost:33300 **user**@**IP-Server**  

Após isso, em um browser, acesse http://localhost:33300 para acessar o Grafana. Em Dashboards, é apresentado os dashboards criados.

<h3 align="center"> 11.6.  Acessando o nosso Prometheus e o Alertmanager. </h3>

Para acessar o nosso Prometheus, a lógica é a mesma do Grafana, fazer o Port Forward.  

* **Inicie o túnel, agora para a porta 39090**: ssh -L 39090:localhost:39090 **user**@**IP-Server**  
* **Crie o port-forward**: kubectl port-forward -n monitoring svc/prometheus-k8s 39090:9090  
* **Após isso, acesse em um browser**: http://127.0.0.1:39090/query  

**Fazendo o Port Forward para o Alertmanager.**  
* **Inicie o túnel, agora para a porta 39093**: ssh -L 39093:localhost:39093 **user**@**IP-Server**  
* **Crie o port-forward**: kubectl port-forward -n monitoring svc/alertmanager-main 39093:9093
* **Após isso, acesse em um browser**: http://127.0.0.1:39093/#/alerts

**Obs**: os comandos para criar o port-forward devem ser realizados no seu servidor e os comandos para criar o túnel devem ser executados no seu host local.  

<h3 align="center"> 11.7. O que é um ServiceMonitor? </h3>

O ServiceMonitor é um recurso customizado (CRD) do Prometheus Operator usado para descobrir e monitorar serviços dentro de um cluster Kubernetes.  
* Em resumo, é parte do ecossistema do Prometheus Operator.  
* Define quais serviços o Prometheus deve monitorar.  
* Substitui o uso do tradicional prometheus.yml (scrape configs) por objetos Kubernetes.  

**Por que usar ServiceMonitor?**  
No Kubernetes com Prometheus Operator, você não edita mais o prometheus.yml diretamente. Em vez disso, você cria recursos ServiceMonitor que informam ao Prometheus: _"Ei, descubra este serviço, nesta porta, neste caminho, com esse intervalo de scraping."_  

**Trabalhando com o ServiceMonitor.**  
Abaixo, comandos para visualizar informações do ServiceMonitor.  
* **Visualizar o ServiceMonitor de um namespace**: kubectl get servicemonitors -n monitoring  
* **Ver os detalhes de um ServiceMonitor**: kubectl get servicemonitors -n monitoring grafana -o yaml
