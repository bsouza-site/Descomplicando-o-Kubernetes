<h2 align="center"> Descomplicando os StatefulSets e os Services no Kubernetes. </h2>

<h3 align="center"> 07.1.	O que é um StatefulSet? </h3>

Os StatefulSets são uma funcionalidade do Kubernetes que gerencia o deployment e o scaling de um conjunto de Pods, fornecendo garantias sobre a ordem de deployment e a singularidade desses Pods.

Diferente dos Deployments e Replicasets que são considerados stateless (sem estado), os StatefulSets são utilizados quando você precisa de mais garantias sobre o deployment e scaling. Eles garantem que os nomes e endereços dos Pods sejam consistentes e estáveis ao longo do tempo.

**O que é um StatefulSet no Kubernetes?**  
StatefulSet é um tipo de controlador usado para rodar aplicações stateful (com estado persistente), onde:

* Cada pod tem identidade fixa;
* Cada pod tem seu próprio volume persistente;
* A ordem de criação e remoção importa;
* Nomes e DNS são previsíveis.

Diferente de Deployments, onde os Pods são descartáveis e idênticos.

Abaixo, todas as configurações explicadas:  

* **apiVersion**: indica a versão da API do Kubernetes usada para criar o recurso. A versão **v1** é a API estável para criar recursos básicos (Pod, Service, ConfigMap).  
* **kind**: define o tipo de recurso que será criado, nesse caso, StatefulSet.  
* **metadata**: informações relacionadas ao StatefulSet.  
* **spec**: nome do Headless service associado.  
  * **serviceName**: nome do serviço.  
  * **replicas**: número de réplicas dos Pods.  
  * **selector**: tudo dentro do bloco selector define quais pods pertencem a esse StatefulSet.  

* **template**: esse bloco define como cada pod será criado. Os labels desse bloco precisam casar com o **selector**.  
* **spec**: define os containers do Pod. O **name** indica o nome interno do container.  
* **ports** e **containerPorts**: definem a porta que o container irá usar.  

* **volumeMounts**: monta o volume persistente no container.  
  * **name**: nome do volume persistente.  
  * **mountPath**: diretório que será montado no container.  

* **volumeClaimTemplates**: define um template de PersistentVolumeClaim. O Kubernetes cria um PVC por pod.
  * **metadata**: indica o name do PVC.

* **spec**: indica as especificações do PVC.  
  * **accessModes**: modo de acesso ao volume. **ReadWriteOnce** indica que o volume pode ser montado por um nó por vez.  
* **resources**:  
  * **requests**:  
  * **storage**: solicita 1Gib de armazenamento.

<h3 align="center"> 07.3.  Criando o nosso Headless Service. </h3>

**O que é Headless Service?**  
Primeiramente, Headless Service no Kubernetes é um Service do Kubernetes sem IP virtual (ClusterIP).  

Ou seja, ele não faz Load Balancer e não esconde os Pods atrás de um único IP, em vez disso, ele expõe diretamente os IPs dos Pods via DNS.  

Para criar um Headless Service, crie o arquivo **07_headless-svc.yml** conforme o exemplo abaixo:
