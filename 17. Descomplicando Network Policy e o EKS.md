<h2 align="center"> Descomplicando Network Policy e o EKS. </h2>

<h3 align="center"> 17.1. Instalando o EKS com o EKSCTL. </h3>

No Kubernetes, uma Network Policy é um conjunto de regras que definem como os Pods podem se comunicar entre si e com outros endpoints de rede. Por padrão, os Pods em um cluster Kubernetes podem se comunicar livremente entre si, o que pode não ser ideal para todos os cenários. As Network Policies permitem que você restrinja esse acesso, garantindo que apenas o tráfego permitido possa fluir entre os Pods ou para/endereços IP externos.

**Para que servem as Network Policies?**  
Networks Policies são usadas para:  
* Isolar Pods de tráfego não autorizado.  
* Controlar o acesso à serviços específicos.  
* Implementar padrões de segurança e conformidade.  

**Conceitos fundamentais: Ingress e Egress.**  
* Ingress: Regras de ingresso controlam o tráfego de entrada para um Pod.  
* Egress: Regras de saída controlam o tráfego de saída de um Pod.  

Ter o entendimento desses conceitos é fundamental para entender como as Network Policies funcionam, pois você precisará especificar se uma regra se aplica ao tráfego de entrada ou de saída.  

**Como funcionam as Network Policies?**  
Network Policies utilizam SELECTORS para identificar grupos de Pods e definir regras de tráfego para eles. A política pode especificar:  

* Ingress (entrada): quais Pods ou endereços IP podem se conectar a Pods selecionados.  
* Egress (saída): para quais Pods ou endereços IP os Pods selecionados podem se conectar.

<h3 align="center"> 17.2. Instalando o Add-on AWS VPC CNI. </h3>

Antes de iniciar a instalação do plugin, verifique em **[Managing-VPC-CNI](https://docs.aws.amazon.com/eks/latest/userguide/managing-vpc-cni.html)** a versão do VPC CNI que devemos instalar em relação a versão do cluster que foi iniciada.
<p align="center"><img width="510" height="291" alt="image" src="https://github.com/user-attachments/assets/39d6af26-6feb-4de6-a965-8d66c8b7a911" /></p>

Para iniciar o cluster, execute: 

**eksctl create cluster --name=eks-cluster --version=1.35 --region=us-east-1 --nodegroup-name=eks-cluster-nodegroup --node-type=t3.medium --nodes=2 --nodes-min=1 --nodes-max=3 --managed**  

Como foi instalada a versão do cluster **1.35**, devemos instalar a versão **1.21.1** do VPC CNI.

Para instalar o plugin **AWS VPC CNI**, execute o comando: **eksctl create addon --name vpc-cni --version v1.21.1-eksbuild.1 --cluster eks-cluster --force**  

Após realizar a instalação desses componentes, acesse o painel do EKS na página da AWS, vá em Add-on e verifique se o componente foi instalado.

Após instalado, clique em Editar, vá em **Valores de configuração** e inclua a configuração **"enableNetworkPolicy": "true"**
<p align="center"><img width="539" height="76" alt="image" src="https://github.com/user-attachments/assets/bfb067cf-c656-4c86-8a73-51e4079bc1f6" /></p>

Após isso, confirme que os pods **aws-node** foram criados com o comando **kubectl get pods -n kube-system**
