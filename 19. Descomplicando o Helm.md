<h2 align="center"> Descomplicando o Helm. </h2>

<h3 align="center"> 19.1. O que é Helm e o Chart? </h3>

O **Helm** é um gerenciador de pacotes para Kubernetes que facilita a definição, instalação e atualização de aplicações complexas no Kubernetes. Com ele você pode definir no detalhe como a sua aplicação será instalada, quais configurações serão utilizadas e como será feita a atualização da aplicação.  

Um **Chart** é um pacote que contém informações necessárias para criar instâncias de aplicações Kubernetes. É com ele que iremos definir como a nossa aplicação será instalada, quais configurações serão utilizadas e como será feita a atualização da aplicação.

Um **Chart**, normalmente é composto por um conjunto de arquivos que definem a aplicação, e um conjunto de templates que definem como a aplicação será instalada no Kubernetes.  

**Artifacthub.**  
Neste contexto de Helm e Chart, um portal muito útil é o **[Artifacthub](https://artifacthub.io/)**. O Artifacthub é um site que centraliza pacotes para Kubernetes, como:

* Helm Charts;  
* Operators;  
* Plugins;  
* etc.   

👉 Pense nele como:  
* App Store do Kubernetes;  
* ou tipo um "Google dos Charts".

<p align="center"><img width="357" height="204" alt="image" src="https://github.com/user-attachments/assets/0b342ff6-ca22-4885-a6ae-0be0982d86c8" /></p>

Utilize o comando **helm lint** para validar se esse chart está correto antes de ser instalado.  

No meu **Chart.yaml**, foi necessário apenas alterar o campo **appVersion** para **appVersion: “1.0”**  

Utilize também o **helm template**. Esse comando serve para renderizar os templates do chart e gerar o YAML final, sem instalar nada no cluster.  

**Obs:** se estiver trabalhando no diretório corrente dos arquivos do Helm, execute os comandos helm lint e helm template com um ponto (.), para entender que o comando deve atuar no diretório corrente.

Agora, devemos utilizar o comando **helm package** para empacotar o Chart em um arquivo **tgz**.

Nesse momento será utilizado o comando **helm repo index**. O comando **helm repo index** no Helm é usado para criar (ou atualizar) o arquivo index.yaml de um repositório de charts.

* **No caso do meu repositório, esse comando será**: helm repo index --url https://github.com/bsouza-site/chart-descom-k8s .  

Agora, execute os comandos abaixo para enviar os arquivos para o Github.
* git add .
* git commit -m "Primeiro commit do nosso repo" -a
* git push origin main

Após isso, no Github, vá em **Settings**, **Pages**, selecione a branch **Main** e clique em **Save**. Copie o endereço que foi gerado (https://bsouza-site.github.io/chart-descom-k8s/)

* **Agora, realize o comando helm repo index novamente, executando**: helm repo index --url https://bsouza-site.github.io/chart-descom-k8s/ .  

A diferença para o primeiro **helm repo** é a nova URL. Agora, realize novamente o commit para o Github com os arquivos atualizados.  

**Adicionando o novo repositório.**  
Agora, iremos adicionar esse novo repositório, para isso, utilize o comando **helm repo add brsouza https://bsouza-site.github.io/chart-descom-k8s/charts**  

**Obs:** talvez pela forma de que os arquivos foram enviados, foi necessário adicionar o **/charts** ao final da URL.  

Após isso, utilize o comando **helm search repo brsouza** para encontrar o meu repositório e ver informações.  

**Instalação.**  
Para instalar o Helm usando o repositório externo, execute os comandos abaixo:

* **Remova o Helm anterior**: helm uninstall giropops-senhas  
* **Instale o novo Helm**: helm install giropops-senhas brsouza/giropops-senhas  

Após esses procedimentos, execute **kubectl get pods** e **kubectl get svc** para ver os componentes implantados.
