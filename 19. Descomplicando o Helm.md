<h2 align="center"> Descomplicando o Helm. </h2>

<h3 align="center"> 19.1. O que é Helm e o Chart? </h3>

O **Helm** é um gerenciador de pacotes para Kubernetes que facilita a definição, instalação e atualização de aplicações complexas no Kubernetes. Com ele você pode definir no detalhe como a sua aplicação será instalada, quais configurações serão utilizadas e como será feita a atualização da aplicação.  

Um **Chart** é um pacote que contém informações necessárias para criar instâncias de aplicações Kubernetes. É com ele que iremos definir como a nossa aplicação será instalada, quais configurações serão utilizadas e como será feita a atualização da aplicação.

Um **Chart**, normalmente é composto por um conjunto de arquivos que definem a aplicação, e um conjunto de templates que definem como a aplicação será instalada no Kubernetes.  

**Artifacthub.**  
Neste contexto de Helm e Chart, um portal muito útil é o **[Artifacthub](https://artifacthub.io/)**. O Artifacthub é um site que centraliza pacotes para Kubernetes, como:

* Helm Charts;  
* Operators;  
* Plugins;  
* etc.   

👉 Pense nele como:  
* App Store do Kubernetes;  
* ou tipo um "Google dos Charts".

<h3 align="center"> 19.3. Instalando a nossa App exemplo. </h3>

Para iniciar os testes com Helm, clone a aplicação de exemplo com o comando:  
* **git clone git@github.com:badtuxx/giropops-senhas-labs.git**  

Acesse o diretório do projeto e aplique os arquivos de configuração com os comandos abaixo:

* kubectl apply -f app-deployment.yaml
* kubectl apply -f app-service.yaml
* kubectl apply -f redis-service.yaml

Verifique com os comandos **kubectl get pods** e **kubectl get svc** se o deployment foi executado corretamente.

Após a aplicação estar online, devemos iniciar a criação do arquivo **Chart.yaml**. Este arquivo é o arquivo principal de metadados de um chart do Helm (o gerenciador de pacotes do Kubernetes).  

Inicie o arquivo **Chart.yaml** com o conteúdo abaixo:  
<p align="center"><img width="445" height="119" alt="image" src="https://github.com/user-attachments/assets/82b44961-a494-4426-a6e3-70739c9a1af3" /></p>

Abaixo, a explicação de cada parâmetro:  
* **apiVersion**: define a versão do formato Chart.  
* **name**: nome o Chart.  
* **description**: uma descrição desse Chart.  
* **version**: a versão desse Chart.  
* **appVersion**: versão da aplicação que o Chart instala.
* **sources**: Lista de URLs com o código-fonte do projeto.  


Agora, crie o diretório **templates**. Nesse diretório, devemos incluir todos os arquivos desse projeto, nesse caso:  

* app-deployment.yaml
* app-service.yaml
* redis-service.yaml
* redis-deployment.yaml

<h3 align="center"> 19.5. Fazendo o deploy do nosso Chart. </h3>

Antes de iniciar o deploy do Chart, realizações as ações abaixo:  

* Crie o diretório **charts**  
* Remova o deployment e o service do giropops-senhas  

* **Para instalar o Chart desse projeto**: helm install giropops-senhas **./**  
<img width="326" height="115" alt="image" src="https://github.com/user-attachments/assets/a7937f06-f9d4-4963-8d6f-68083e9521d6" />

Após isso, pode executar **kubectl get pods** e **kubectl get svc** que os serviços foram implantados.  

**Abaixo, os principais comandos do Helm:**  
* **Listar os helm implantados**: helm list  
* **Desinstalar um Helm**: helm uninstall <helm>  
* **Para ver mais detalhes do helm instalado**: helm get all giropops-senhas
